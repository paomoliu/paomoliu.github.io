SourceTree成功访问SVN

  • SourceTree SVN git

个人比较喜欢使用SourceTree，以前从未使用它访问过SVN。由于实习，公司使用的SVN管理项目源码，因而有了SourceTree访问SVN的尝试。在这个过程中遇到很多的坑，所以做个总结，方便自己以后查看。

SourceTree简介

SourceTree是Windows和Mac OS下免费的git和hg客户端，同时也是Mercurial和Subversion版本控制系统工具。支持创建、克隆、提交、push、pull和合并等操作。

SourceTree安装

不建议直接从App Store搜索下载SourceTree，因为App Store里的版本太低。建议直接前往官网下载。安装步骤如下：

  1. 下载最新版SourceTree；
  2. 双击打开压缩包的.dmg文件，将SourceTree的App应用拖放到Applications里；
  3. 在Lanchpad中找到SourceTree并打开它；
  4. 接受协议，Continue；
  5. 设置指定的账户登录（我一般使用github帐号进行登录），也可以直接Skip该步骤；
  6. 安装成功。

SourceTree访问SVN地址

SourceTree一般是与github等进行连接，若安装好之后，立即使用它访问SVN地址，会出现不能正常检测SVN地址的问题，所以必须得做一些配置。步骤如下：
1）终端命令：svn ls https://192.168.0.24:8443/XXXXX/（根据不同的SVN地址代替XXXXX部分），回车；
2) 查看回车后终端输出数据，在最后一行，选择部分，输入p，回车；
3)输入Mac密码。

    3）等待软件安装完成，如图所示：
